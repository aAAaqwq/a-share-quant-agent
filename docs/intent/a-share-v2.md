# 意图确认书 — A股量化系统 v2

> 状态: ✅ 已确认 (2026-07-02)
> 方法: interview-me 逐问澄清,用户逐行确认 + 明确 yes
> 下游: 方案设计 → 分阶段实现

## 一句话

把项目从「亏钱的批量回测 CLI」改造成 **盘前 agent 定「主力板块+主力方向+15 只候选涨停」→ 竞价/盘中确定性脚本实时验证 → 人看 dashboard 决策** 的**研究型决策支持系统**,目标是**找到板块级 edge**(当前无 edge)。

## 逐行意图

| 维度 | 结论 |
|------|------|
| **产出物** | 盘前 agent 预测 → 竞价/盘中实时验证 → dashboard 决策支持系统 |
| **用户** | 用户本人。盘前看 agent 分析,盘中盯 dashboard,**最终决策人工拍板**(不自动下单) |
| **为什么现在** | 现有 5 因子死打分**无 edge**(累计 -5.98%);靠 agent 逻辑/因果分析 + 竞价验证系统性搜 edge |
| **成功标准** | 主看**「主力板块/方向」判断准确率**(D 双轨:9:25 竞价当场 + 收盘复核);15 名单竞价兑现率为辅。**不再以个股 P&L 为主指标** |
| **数据准确度(top 优先级)** | 校验层 + 多源交叉 + 剔除过期/虚假/无效;竞价分板块数据源**可插拔** |
| **实时性** | 「实时」= 确定性脚本刷新,**非 agent 实时**;agent 盘前只跑一次 |
| **SKILL.md** | 写成**指导 agent 跑盘前分析流程**的操作手册 |

## 时间轴 + cadence

```
盘前 (8:30, 晚点无所谓)   → GitHub Actions: agent 重分析 → JSON(板块/方向/15名单)
集合竞价 (9:15–9:25)       → 常驻脚本 30s 高频: 拉竞价数据 → 双轨打分 → 推 Cloudflare KV
盘中 (9:30–15:00)         → 常驻脚本 1h 一次: 低频刷新
收盘复核 (15:30, 低频)    → GitHub Actions: 双轨记分 + 更新统计
展示                      → Cloudflare Pages 只读 KV, 顶部 last_update 死活指示(>60s 变红)
```

## 计算落点

- **一个可移植拉数据脚本** `intraday_puller.py`,config 控制 cadence + 推送目标,**本地 / VPS 同一份代码零改动**。
- 盘前重分析 + 收盘复核 → GitHub Actions。
- 竞价 30s 高频 → **必须常驻 + 准点**(VPS 首选 systemd 9:14 拉起;本地备选,只需 9:14–9:26 开机)。
- **物理限制**: GitHub Actions cron 不准点,无法承接竞价 30s 高频,躲不掉常驻宿主。
- 可观测性: dashboard 顶部 `last_update` 时间戳,超 60s 变红。

## 明确不做 (Out of scope)

- 自动下单 / 实盘交易(人决策)
- 个股 P&L 调优(已明确不是重点)
- 花哨可视化(先能看清数据变化即可)
- 盘中让 agent 滚动重跑
